> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**目标：** 将 `upstream/main` 的 v0.1.151 更新合并到本地定制版本，完成逻辑审查、回归验证、Git 推送和新正式 VPS 镜像化上线。

**架构：** 从 `dev` 创建独立上游同步分支，保留上游合并提交并逐项审查与本地定制交叉的 OpenAI 转发和设置逻辑。验证通过后先合并回 `dev`，再按用户本次明确的正式上线授权合并到 `main`，由新正式 VPS 从 GitHub 拉取同一提交构建镜像并切换生产。

**技术栈：** Go 1.26、Gin、Ent、Vue 3、TypeScript、Vitest、Docker Compose。

## 全局约束

- 禁止覆盖用户已有改动；每次修改前后检查 `git status --short` 和 diff。
- 不猜测接口、字段、配置和线上状态，所有结论必须来自代码、测试或服务器只读检查。
- 新正式 VPS 只从 GitHub 拉取已提交源码，使用 `deploy/Dockerfile` 构建带 commit 的镜像。
- 发布前记录旧镜像和数据状态，发布后验证健康检查、管理端账号接口、购买页、模型页、日志和数据库连接。
- 本次上游没有新增数据库迁移；若合并结果与该结论不一致，必须停止发布并重新评估备份和回滚。
