# 创意绘图功能说明

## 功能入口

创意绘图入口位于用户侧左侧菜单 `创意绘图`，路由为 `/creative-drawing`。用户登录后可进入页面，使用自己账号下绑定 OpenAI 分组的 API 密钥调用本系统图片网关。

## 使用角色与前置条件

- 使用角色：普通用户与管理员的个人账户区均可使用。
- 用户必须已登录。
- 用户至少需要一个状态为启用、已绑定 OpenAI 平台分组且分组允许图片生成的 API 密钥。
- 分组和上游账号需要支持图片生成能力；若上游不支持，网关会返回对应错误。

## 操作流程

1. 用户进入 `创意绘图` 页面。
2. 系统加载用户 API 密钥列表，并默认选择第一个启用且绑定 OpenAI 分组的密钥。
3. 页面采用左右分栏操作台：左侧为功能区，右侧为图片展示区。
4. 用户在左侧功能区输入画面描述，也可以打开 `提示词市场` 选择提示词。
5. 用户在左侧功能区选择 API 密钥、生成数量、输出格式、快捷尺寸、尺寸模式、画幅比例、分辨率或手动宽高；创意绘图固定使用 `gpt-image-2`，不再暴露对话模式或模型选择。
6. 快捷尺寸与模型测试台保持一致：Auto、1024×1024 · 1K、1536×1024 · 2K、1024×1536 · 2K、2048×2048 · 2K、3840×2160 · 4K、2160×3840 · 4K。
7. 用户可在左侧功能区上传参考图，或从热门模板带入参考图 URL。模板案例存在 `reference_image_urls` 时会将这些图片作为参考图；编辑类案例没有显式参考图时会使用预览图作为参考图；普通文生图案例没有显式参考图时不会把成品预览误用为参考图。远程参考图会先在左侧功能区显示预览，前端可访问时会转为本地参考图后再提交。
8. 用户点击 `开始生成` 后：
   - 前端先请求 `POST /api/v1/creative-drawing/tasks` 创建持久任务。
   - 后端任务 worker 使用用户选择的 API Key 继续调用图片网关。
   - 无参考图时，worker 请求 `/v1/images/generations`。
   - 有参考图时，worker 请求 `/v1/images/edits`。
9. 请求进入生成队列后，左侧功能区会清空当前输入内容和参考图；本轮提示词、参数、参考图快照和任务 ID 会保存在当前会话轮次中。
10. 页面会轮询任务状态；即使页面刷新、浏览器关闭或网络中断，后端任务仍会继续执行并把成功结果或失败原因写入数据库。
11. 用户重新打开页面时，前端会读取最近的后端任务并同步回本地会话。左侧预计消耗会按当前尺寸对应的 1K/2K/4K 层级展示。生成结果展示在右侧当前创作会话中，并保存到浏览器本地历史。
12. 图片生成完成后，右侧结果区支持预览、一键下载、重试本轮提示词、把结果加入左侧参考图继续创作，以及打开图片原始地址。

## 涉及接口

- `GET /api/v1/keys`：读取当前用户 API 密钥，用于选择自己的调用密钥。
- `POST /api/v1/creative-drawing/tasks`：创建创意绘图持久任务，返回任务 ID 和初始状态。
- `GET /api/v1/creative-drawing/tasks`：读取当前用户最近的创意绘图任务，用于页面恢复和状态同步。
- `GET /api/v1/creative-drawing/tasks/:id`：读取单个创意绘图任务状态和结果。
- `POST /v1/images/generations`：通过 Sub2API 图片生成网关提交文生图请求。
- `POST /v1/images/edits`：通过 Sub2API 图片编辑网关提交参考图作画请求。

创意绘图任务只保存用户选择的 API Key ID，不把 API Key 明文写入任务表。worker 执行时会重新按用户所有权读取密钥，再使用该 API Key 作为 `Authorization: Bearer <key>` 调用图片网关，仍沿用现有网关鉴权、分组调度、计费、限额和使用记录逻辑。

图片生成和编辑请求默认要求上游返回 `b64_json`。当 OpenAI APIKey 类型上游兼容服务仅返回受保护的 `url`、`image_url` 或 `download_url` 时，网关会使用同一次上游请求的鉴权头在服务端下载图片内容，并补齐 `data[].b64_json`，避免浏览器直接访问上游受保护图片地址导致结果无法显示。

图片上游返回 `502`、`503` 或 `504` 这类临时不可用错误时，网关会先在同一个已选账号上最多重试 3 次；参数错误、鉴权错误和限流错误不走该兜底，避免掩盖真实配置或请求问题。池模式账号仍保留原有 `401`、`403`、`429` 同账号重试策略。

参考图作画任务默认以流式图片编辑请求提交到网关，并请求上游返回最多 2 张中间进度图；worker 只持久化最终完成图。这样可以让 4K 参考图作画在长时间执行时持续保持上游数据流，减少非流式请求长时间无响应导致的 `504`。

流式图片编辑返回错误事件时，worker 会识别 `type=error`、`response.failed` 以及顶层 `error.message` 等常见格式；即使上游以 HTTP 200 返回错误 JSON，也会优先提取真实错误原因并写入任务错误信息，避免误显示为“图片接口没有返回可展示的图片”。

图片编辑请求支持参考图随任务持久化：

- 浏览器本地参考图、页面预设图和已加载的远程参考图会随任务以 `data:image/...` 形式保存到 `creative_drawing_tasks.reference_images`，因此任务执行不依赖页面仍然打开。
- 参考图进入网关后会按上游账号类型归一化：
  - 浏览器本地上传图、页面预设图优先以 `data:image/...` 形式提交。
  - 远程参考图使用 JSON `images[].image_url` 提交到网关。
  - 当上游账号为 OpenAI APIKey 类型时，网关会先校验远程图片地址必须为公网 HTTP/HTTPS 地址，再下载图片并转成 multipart 文件上传给上游。
  - 当上游账号为 OpenAI OAuth 类型时，网关同样会把远程参考图下载并转成 `data:image/...` 后再送入 Responses 图片工具，避免热门模板预设图因远程 URL 访问差异导致编辑作画失败。
- OAuth 图片链路会兼容 Responses 返回的 `result`、`b64_json`、`download_url`、`url`、`image_url` 等图片资产字段，避免上游已经生成图片但本地误判为失败。
- OAuth 图片链路若只拿到上游 `download_url` 或 `url`，网关会使用同一上游账号、请求头和代理先下载图片，再转换为 `b64_json` 返回给前端，避免浏览器因缺少上游会话或签名过期显示坏图。

## 热门模板

热门模板远程读取以下公开数据源：

- `glidea/banana-prompt-quicker`
- `EvoLinkAI/awesome-gpt-image-2-prompts`

热门模板支持来源、语言、分类、模式、NSFW 和收藏筛选。收藏内容保存在浏览器本地，不写入数据库。

## 本地数据

以下数据保存在浏览器本地：

- 创作会话历史
- 当前选中会话
- 热门模板收藏

创作会话历史的轻量元数据保存在 `localStorage`；生成图的 base64 正文会优先写入浏览器 IndexedDB，并在读取历史时恢复为可展示 `data:image/...` 地址。这样可以避免 2K/4K 大图直接写入 `localStorage` 触发浏览器配额，导致完成后的历史仍停留在“生成中”或图片丢失。

新提交的创意绘图轮次会带有后端任务 ID。页面刷新、浏览器关闭或网络中断后，重新进入页面会通过任务 ID 或任务列表查询后端真实状态：

- 任务仍在 `queued` / `running`：继续显示生成中并保持轮询。
- 任务为 `success`：同步图片结果并写入本地 IndexedDB。
- 任务为 `error`：同步失败原因并展示在对应轮次。

旧版本本地历史里没有任务 ID 的生成中轮次无法从后端恢复，仍会标记为中断错误，提示用户重新发起创作。

后端新增 `creative_drawing_tasks` 表保存任务元数据、参考图快照、执行状态、错误信息和图片结果。图片结果当前以任务 JSON 形式持久化；后续如图片量继续增大，可再迁移到文件或对象存储。

创意绘图后端 worker 单个任务最长运行 30 分钟。这个窗口用于覆盖 4K 参考图作画在图片网关内的上游长耗时和同账号重试；超过该时间仍未完成时，任务会被标记为超时错误。

清空浏览器站点数据会删除这些本地记录，不影响服务器上的 API 密钥、余额、用量日志和分组配置。

## 异常与边界处理

- 未输入提示词时阻止提交。
- 没有可用 OpenAI 分组密钥时提示用户先创建或绑定密钥。
- 请求进入生成队列后会清空左侧功能区输入和参考图，但当前轮次会保留提交时的提示词和参考图快照。
- 提示词市场带入的远程参考图提交前会先完成加载；存在加载失败的参考图时会提示用户移除失败图片或重新套用，避免把无效参考图继续提交到图片编辑接口。
- 图片生成失败时在当前创作历史中保留失败记录和错误信息，便于用户调整参数后重试。
- 右侧结果区的 `重试` 只会把提示词、尺寸、数量、格式和参考图回填到左侧功能区；用户仍需再次点击 `开始生成`，避免误触重复扣费。
- 右侧结果区的 `加入参考图` 会把当前结果图转成参考图回填到左侧功能区；后续再次提交时自动走图片编辑接口。
- 参考图最多保留 8 张，避免浏览器本地存储和请求体过大。
- 图片尺寸会按 16 像素倍数、最大边、像素总量和宽高比限制进行归一化。
