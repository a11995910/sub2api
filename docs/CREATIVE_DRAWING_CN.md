# 创意绘图功能说明

## 功能入口

创意绘图入口位于用户侧左侧菜单 `创意绘图`，路由为 `/creative-drawing`。用户登录后可进入页面，使用自己账号下绑定 OpenAI 分组的 API 密钥调用本系统图片网关。

## 使用角色与前置条件

- 使用角色：普通用户与管理员的个人账户区均可使用。
- 用户必须已登录。
- 用户至少需要一个状态为启用、已绑定 OpenAI 平台分组且分组允许图片生成的 API 密钥。
- 分组和上游账号需要支持图片生成能力；若上游不支持，网关会返回对应错误。

## 操作流程

1. 用户进入 `创意绘图` 页面。
2. 系统加载用户 API 密钥列表，并默认选择第一个启用且绑定 OpenAI 分组的密钥。
3. 用户可以直接输入画面描述，也可以套用首页预设或打开 `Prompts 提示词市场` 选择提示词。
4. 用户可上传参考图，或从提示词市场带入参考图 URL。市场案例存在 `reference_image_urls` 时会将这些图片作为参考图；编辑类案例没有显式参考图时会使用预览图作为参考图；普通文生图案例没有显式参考图时不会把成品预览误用为参考图。
5. 用户在参数面板中选择密钥、模型、数量、尺寸模式、比例、分辨率和输出格式。
6. 用户点击发送后：
   - 无参考图时，请求 `/v1/images/generations`。
   - 有参考图时，请求 `/v1/images/edits`。
7. 生成结果展示在当前创作会话中，并保存到浏览器本地历史。

## 涉及接口

- `GET /api/v1/keys`：读取当前用户 API 密钥，用于选择自己的调用密钥。
- `POST /v1/images/generations`：通过 Sub2API 图片生成网关提交文生图请求。
- `POST /v1/images/edits`：通过 Sub2API 图片编辑网关提交参考图作画请求。

图片网关接口使用用户自己的 API Key 作为 `Authorization: Bearer <key>`，仍沿用现有网关鉴权、分组调度、计费、限额和使用记录逻辑。

## 提示词市场

提示词市场远程读取以下公开数据源：

- `glidea/banana-prompt-quicker`
- `EvoLinkAI/awesome-gpt-image-2-prompts`

市场支持来源、语言、分类、模式、NSFW 和收藏筛选。收藏内容保存在浏览器本地，不写入数据库。

## 本地数据

以下数据保存在浏览器本地：

- 创作会话历史
- 当前选中会话
- 提示词收藏

清空浏览器站点数据会删除这些本地记录，不影响服务器上的 API 密钥、余额、用量日志和分组配置。

## 异常与边界处理

- 未输入提示词时阻止提交。
- 没有可用 OpenAI 分组密钥时提示用户先创建或绑定密钥。
- 图片生成失败时在当前会话中保留失败轮次和错误信息，便于用户调整参数后重试。
- 参考图最多保留 8 张，避免浏览器本地存储和请求体过大。
- 图片尺寸会按 16 像素倍数、最大边、像素总量和宽高比限制进行归一化。

## 关联模块

- 用户 API 密钥管理：`/keys`
- OpenAI 图片网关：`/v1/images/generations`、`/v1/images/edits`
- 使用记录与计费：沿用现有网关使用记录、图片计费和 API Key 限额逻辑
- 前端菜单与路由：用户侧导航、页面标题和国际化文案
