# 灵石充值、订阅兑换与模型广场说明

## 功能范围

用户侧核心入口：

- `灵石充值`：用户在 `/purchase` 页面内嵌外部小铺购买余额兑换码，购买后前往 `/redeem` 兑换灵石余额。
- `我的订阅`：用户在 `/subscriptions` 页面查看订阅用量，并直接使用灵石兑换可售订阅套餐。
- `模型广场`：用户在 `/models` 查看当前可调用模型、可用分组，以及按用户有效倍率计算后的灵石价格。
- `模型测试台`：用户在 `/model-test` 选择模型、分组和对应 API Key，通过真实 `/v1` 网关在线测试文本或图片模型。

用户侧菜单已不再展示“可用渠道”和“渠道状态”入口，相关路由保留，避免影响已有接口、调度逻辑和管理员能力。

## 充值页

充值页文件：`frontend/src/views/user/RechargeRedeemView.vue`

外部小铺地址固定为：

```text
https://pay.ldxp.cn/shop/R5D7AG9X
```

页面提供：

- 内嵌 iframe 购买入口。
- 新窗口打开小铺入口。
- 当前灵石余额展示。
- 跳转兑换页 `/redeem` 的按钮。

充值页只展示灵石充值相关内容，不再承载订阅购买入口。订阅套餐统一在“我的订阅”页面用灵石兑换。

后端 CSP 已在 `backend/internal/server/middleware/security_headers.go` 中把 `https://pay.ldxp.cn` 加入 `frame-src`。如果目标站点自身后续增加禁止嵌入的响应头或 CSP，iframe 仍可能空白；此时用户可使用“新窗口打开”购买兑换码，再回到兑换页使用。

## 余额兑换码与邀请返利

用户在 `/redeem` 使用 `balance` 类型且面额大于 0 的兑换码时，系统会先在数据库事务中标记兑换码已使用并增加用户灵石余额。兑换成功后，系统按当前邀请返利策略为该用户绑定的上游邀请人累计返利额度，并在返利流水中记录来源兑换码。

兑换码返利遵循邀请返利的统一规则：

- `affiliate_enabled` 关闭时不产生新返利。
- 用户未绑定上游邀请人时不产生返利。
- 返利金额按邀请人的专属返利比例优先计算；未设置专属比例时使用全局返利比例。
- 配置返利有效期时，超过被邀请用户注册后的有效期不再产生返利。
- 配置单个被邀请人返利上限时，系统会按剩余额度截断返利金额。
- 配置冻结期时，返利先进入冻结额度，到期后再解冻为可提额度。

支付订单充值仍由订单履约流程发放邀请返利，并使用订单来源记录返利流水。订单履约中自动使用的充值兑换码会跳过兑换层返利，避免同一笔支付重复返利。

管理端邀请返利的“返利记录”展示两类来源：

- `订单`：来源于内置支付订单充值，显示订单号、外部交易号、支付方式和订单状态。
- `兑换码`：来源于用户站内兑换余额兑换码，显示兑换码 ID、兑换码内容、兑换金额和兑换状态。

## 订阅兑换

订阅页面文件：`frontend/src/views/user/SubscriptionsView.vue`

后端接口：

```text
POST /api/v1/payment/plans/:id/purchase-with-balance
```

兑换流程：

- 前端读取 `/api/v1/payment/plans` 展示所有可售订阅套餐，保留套餐原价格、划线原价、描述、有效期、特性和关联分组信息。
- 用户点击“用灵石兑换”后会先看到确认弹窗；用户确认后，前端才调用兑换接口。
- 后端校验套餐仍可售、分组仍为启用的订阅分组、用户状态正常且灵石余额充足。
- 后端在同一数据库事务中扣减用户灵石、生成一张 `subscription` 类型兑换码、标记兑换码已使用，并创建或续期对应用户订阅。
- 兑换码面额记录为本次套餐价格，备注会写入套餐名称和套餐 ID，管理员后续可以在兑换码记录中查看这笔订阅兑换。
- 用户余额不足时不会创建兑换码，也不会修改订阅。

## 模型广场

模型广场文件：`frontend/src/views/user/ModelMarketView.vue`

页面复用现有用户接口，不新增后端接口：

- `/api/v1/channels/available`：读取用户可访问分组与支持模型定价。
- `/api/v1/groups/rates`：读取用户专属分组倍率。

页面展示改为“分组优先”：

- 顶部横向展示当前用户可访问的分组，支持按分组名、平台名和模型名搜索。
- 用户选中一个分组后，下方只展示该分组可调用的模型卡片。
- 模型卡片展示平台、计费模式、阶梯数量、当前分组倍率和倍率后的灵石价格。
- 页面不再展示“最优价格”或跨分组价格对比，避免在新增分组渠道后误导用户。

价格展示规则：

- 基础定价来源于渠道模型定价配置。
- 文本、Token、按次价格使用用户有效文本倍率：优先用户专属倍率，其次分组默认倍率。
- 图片价格在分组启用独立图片倍率时使用图片倍率，否则使用文本倍率。
- 图片生成能力作为独立模型展示：OpenAI 图片分组会生成 `image-2` 条目，普通 GPT 模型只展示非图片分组，避免把 `ChatGPT2API 图片` 这类图片分组挂到所有 token 模型下。
- 用户余额、实际扣费、充值、订阅兑换和模型价格展示为灵石；用量仪表盘和使用记录中的原价/标准成本用于对照上游原始成本，展示为美元。

图片模型卡片会直接按当前分组展示 `1K / 2K / 4K` 三档图片价格；未配置的档位展示 `-`，真实扣费仍按后端图片计费回退规则处理。每个模型卡片提供“去测试”按钮，会跳转到模型测试台并预选对应模型与分组。

模型广场只展示模型、平台、分组、价格和测试入口，不展示渠道名称、渠道状态或上游账号状态，避免把调度细节暴露给普通用户。

## 模型测试台

模型测试台文件：`frontend/src/views/user/ModelTestView.vue`

页面复用现有用户接口，不新增后端测试接口：

- `/api/v1/channels/available`：读取用户可见模型、可用分组和图片价格。
- `/api/v1/groups/rates`：读取用户专属分组倍率。
- `/api/v1/keys`：读取当前用户 active API Key，并按所选分组筛选可用 Key。
- `/v1/chat/completions`：文本模型测试端点。
- `/v1/images/generations`：图片模型测试端点。
- `/v1/images/edits`：上传参考图后的图片编辑测试端点。

测试请求使用用户选中的真实 API Key 发起，因此会自然经过 API Key 鉴权、分组路由、限流、用量记录和灵石扣费。若某个分组没有 active API Key，页面只提示用户去 API Key 页面创建或绑定该分组的 Key，不会用无分组 Key 代替，避免测试流量落到错误分组。

图片测试支持固定尺寸和自适应尺寸。固定尺寸会向图片网关传递 `size`；自适应尺寸不会传递 `size`，适用于用户在提示词中自行描述尺寸、画幅或比例的场景。自适应尺寸的价格预览按后端当前默认 `2K` 分档展示，真实请求仍经过网关的图片计费记录链路。

图片模式可上传参考图片。未上传参考图时调用 `/v1/images/generations` 生成新图；上传本地参考图后调用 `/v1/images/edits`，以 multipart 表单提交 `model`、`prompt`、可选 `size`、`n=1`、`response_format=b64_json` 和图片文件。远程参考图可通过 JSON `images[].image_url` 提交到网关；当上游账号为 OpenAI APIKey 类型时，网关会校验远程地址为公网图片后转成 multipart 文件上传给上游。前端限制最多 4 张参考图，单张不超过 20MB，与后端图片上传单文件限制保持一致。

## 货币单位

前端统一通过 `formatSpiritStones()` 或 `common.currencyName` 展示余额、用量、兑换码面额和后台金额字段。历史字段名如 `*_usd` 暂不变更，原因是它们已经参与接口、数据库字段、统计逻辑和旧数据兼容；本次仅调整用户可见单位文案。

## 影响范围

- 用户侧：菜单、充值页、兑换入口、模型广场、模型测试台、余额和用量展示。
- 管理侧：用户余额、分组额度、订阅额度、兑换码、邀请返利记录、推广码、订单、统计图表等金额展示文案。
- 后端：余额兑换码成功兑换后纳入邀请返利；支付订单返利继续走订单履约链路；返利流水保留订单来源和兑换码来源，便于后台审计；同时保留充值页 iframe 所需的 CSP 白名单，不改变渠道调度逻辑。
