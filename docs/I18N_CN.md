# 前端国际化

## 运行时语言包装配

前端支持 `zh` 与 `en` 两种语言。`frontend/src/i18n/index.ts` 按需加载当前语言，并将结果注册到 Vue I18n。

每种语言由两层消息组成：

- `frontend/src/i18n/locales/<locale>/index.ts` 聚合上游按业务域拆分的语言包，用于持续接收上游新增页面和字段文案。
- `frontend/src/i18n/locales/<locale>.ts` 保存本项目签到、模型广场、灵石购套餐等定制功能文案，并覆盖拆分语言包中的同名路径。

两层消息通过 `mergeLocaleMessages` 递归合并。对象节点逐层合并，数组与普通值整体覆盖，因此拆分语言包新增的键不会被定制语言包中较旧的同级对象遮蔽，已有定制措辞仍保持优先。

动态导入必须显式指定拆分语言包的 `index` 与定制语言包的 `.ts` 文件，不使用 `./locales/zh` 或 `./locales/en` 这类同时匹配文件和目录的模糊路径。

## 异常与边界

- 浏览器保存的语言值仅接受 `zh`、`en`；其他值回退到浏览器语言或英文。
- 当前语言包加载完成前不注册空的占位消息，加载失败由应用启动错误链路处理。
- 找不到翻译键时 Vue I18n 会显示键名，因此新增页面文案必须同时补充中英文键。
- 定制语言包与拆分语言包存在同名叶子键时，以定制语言包为准。

## 自动化验证

执行以下测试可验证语言包聚合键无冲突、合并结果覆盖两层语言包全部路径，以及管理概览和用量页关键中文文案不会直接显示翻译键：

```bash
cd frontend
pnpm exec vitest run src/i18n/__tests__
```

修改语言包装配方式、目录结构或覆盖顺序时，必须同步更新本文档及 `localeMessageMerge.spec.ts`。
